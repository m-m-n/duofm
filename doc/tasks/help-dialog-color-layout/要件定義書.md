# 要件定義書: ヘルプダイアログのカラーパレット表示レイアウト改善

## 概要

ヘルプダイアログ内のカラーパレット表示において、1行あたりの色数が多すぎてダイアログの幅を超えてしまい、表示が崩れる問題を修正する。

## 現状の問題

- カラーキューブ（16-231）: 1行6色で表示
- グレースケール（232-255）: 1行6色で表示
- 各色は `番号=HEX値` 形式（例: `16=#000000`）で約15文字
- 1行あたり約90文字となり、ダイアログ幅（70文字）を超過
- 結果としてダイアログが横に伸びて不恰好になる

## 要件

### 機能要件

- FR1.1: カラーキューブ（16-231）の表示を1行2色に変更する
- FR1.2: グレースケール（232-255）の表示を1行2色に変更する
- FR1.3: 各色の表示フォーマット `番号=HEX値` は維持する
- FR1.4: 標準色（0-15）の表示は現状維持（1行8色）

### 非機能要件

- NFR1.1: ダイアログ幅（70文字）内に収まること

## 対象外

- 標準色（0-15）のレイアウト変更

## 成功基準

- [ ] カラーキューブが1行2色で表示される
- [ ] グレースケールが1行2色で表示される
- [ ] ダイアログが横に伸びない
- [ ] 既存のテストが通過する
